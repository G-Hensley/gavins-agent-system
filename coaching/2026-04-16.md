### 2. Add `--verify` + symlink preflight to the install, then gate commits on it

- **Problem:** The install silently half-completes on Windows — backups succeed, `ln -s` fails under MSYS without Developer Mode/admin, and the script keeps going. You only notice when a skill you know you added isn't triggering. This is exactly the "sharp edge users find on first use" friction.

Exact text to add after line 14 (the `MSYS2_ARG_CONV_EXCL` export):
  ```bash
  # Preflight: can we actually create symlinks on this host?
  if command -v cygpath >/dev/null 2>&1; then
    _tmpt="$(mktemp -u)"; _tmpl="${_tmpt}.lnk"
    if ! ln -s "$_tmpt" "$_tmpl" 2>/dev/null; then
      echo "ERROR: cannot create symlinks. Enable Windows Developer Mode" >&2
      echo "       (Settings > For developers > Developer Mode) or re-run in" >&2
      echo "       an Administrator Git Bash shell." >&2
      exit 2
    fi
    rm -f "$_tmpl"
  fi
  ```
