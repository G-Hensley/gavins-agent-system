### Tier 4 — rubric staleness, not behavior regression

The `task-manager-app` eval would likely still mechanically pass against the **existing** rubric. However, the rubric no longer adequately tests the system because it predates the orchestration arc (PRs #14–22).

The eval rubric does **not** check for any of these recently-shipped patterns:

| Pattern | Shipped in | Rubric coverage |
|---|---|---|
| Per-task PR rhythm in `subagent-driven-development` Stage 7 | PR #15 | Missing |

The eval would pass mechanically because the old patterns still work. But it would no longer verify that the new orchestration-era patterns are followed — meaning the rubric has lost signal value for what's actually being tested in real work.

**Verdict: mechanical re-run not worth $30–100+ until the rubric is refreshed. The result against the old rubric tells us nothing new.**

### Cross-cutting — hook output channel

The PostToolUse hooks (`lint-on-save`, `doc-drift-check`, `pr-push-nudge`, `file-size-cap`, `verify-tests`) now emit JSON `additionalContext` that reaches the assistant (PRs #9, #10). Evals dispatch agents which may trigger these hooks. The JSON channel surfaces warnings the agents previously missed silently.

**Effect on evals:** likely positive — agents now get warnings they used to miss. No eval rubric currently checks hook-fed context. Could be worth adding to a future rubric.
